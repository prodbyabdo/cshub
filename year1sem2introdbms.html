Course Summary

  • DBMS: A software system that manages, stores, and retrieves data efficiently in a structured format.
  • Problems with file-based systems: Data Redundancy, Inconsistency, Difficult Access, Poor Security, Single-User Access, and No Backup/Recovery.
  • Six key components: Hardware, Software, Data, Procedures, Database Access Language, and People.
  • Operational Data vs Metadata: Operational Data is actual user data; Metadata is data about data.
  • Types of DBMS: RDBMS, NoSQL DBMS, and Cloud-Based Database.
  • Languages: DDL, DML, DCL, plus DQL for retrieval and TCL for transaction control.
  • Abstraction levels: Internal (Physical), Conceptual (Logical), External (View).
  • Normalization: Organizing data to reduce redundancy and anomalies.
  • Normal forms: 1NF (atomic values), 2NF (remove partial dependency), 3NF (remove transitive dependency), BCNF (every determinant is a candidate key).
