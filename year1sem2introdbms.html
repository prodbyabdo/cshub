Subject Summary

DBMS Basics

  • A DBMS is software that stores, organizes, secures, and retrieves data efficiently.
  • Core goals include reduced redundancy, improved integrity, data sharing, and backup/recovery support.
  • Examples include MySQL, PostgreSQL, Oracle Database, SQL Server, and MongoDB (NoSQL model).

Database Types

  • Relational databases: table-based schema, SQL querying, strong consistency.
  • Hierarchical/network models: parent-child or graph-like record relationships.
  • NoSQL databases: key-value, document, column-family, and graph systems for scale and flexibility.
  • Centralized vs distributed: single-site control versus replicated/partitioned multi-site systems.

SQL Language Categories

  • DDL: CREATE, ALTER, DROP for schema definition.
  • DML: INSERT, UPDATE, DELETE for data manipulation.
  • DQL: SELECT for querying data.
  • DCL/TCL: GRANT, REVOKE, COMMIT, ROLLBACK for control and transactions.

Abstraction Levels

  • Physical level: storage structures, indexing, access paths.
  • Logical level: entities, relationships, constraints, schema design.
  • View level: user-specific virtual views and restricted presentation of data.

Normalization

  • 1NF: remove repeating groups, enforce atomic values.
  • 2NF: remove partial dependency on composite keys.
  • 3NF: remove transitive dependency and keep non-key attributes dependent on the key.
  • Normalization improves consistency and update reliability while balancing performance needs.

Short-Answer Key (20)

  1. A DBMS is software for creating and managing databases. Advantages include reduced redundancy and improved security/data sharing.
  2. A file system stores isolated files with duplication risk, while a DBMS provides centralized control, querying, concurrency, and integrity constraints.
  3. A primary key uniquely identifies each record in its own table. A foreign key references a primary key in another table to enforce relationships.
  4. CREATE TABLE Student (RollNo INT PRIMARY KEY, Name VARCHAR(50), Department VARCHAR(30));
  5. Normalization organizes data to reduce redundancy and update anomalies, improving consistency and maintainability.
  6. 1NF uses atomic fields; 2NF removes partial dependency on composite keys; 3NF removes transitive dependency on non-key attributes.
  7. ACID means Atomicity, Consistency, Isolation, and Durability, ensuring reliable transaction processing.
  8. DDL defines schema (CREATE), DML modifies data (UPDATE), DQL queries data (SELECT), DCL controls privileges (GRANT), TCL manages transactions (COMMIT).
  9. Schema is the structural blueprint of database objects. Instance is the current data state at a specific time.
  10. INNER JOIN returns only matching rows; LEFT JOIN returns all left-table rows plus matching right-table rows.
  11. An index is a fast lookup structure on columns. Benefit: faster search. Drawback: extra storage and slower writes.
  12. Data abstraction hides complexity at three levels: physical, logical, and view.
  13. Data integrity ensures correctness and consistency. Entity integrity requires non-null unique primary keys, and referential integrity requires valid foreign-key references.
  14. Centralized databases are managed at one location; distributed databases spread data across multiple sites with coordination.
  15. A view is a virtual table based on a query; it simplifies reporting and restricts sensitive data exposure.
  16. INSERT INTO Student (RollNo, Name, Department) VALUES (101, 'Amina', 'CS');
  17. UPDATE Student SET Department = 'IT' WHERE RollNo = 101;
  18. SELECT Department, COUNT(*) AS total_students FROM Student GROUP BY Department;
  19. Backup creates restorable copies of data; recovery restores database consistency after failure.
  20. Concurrency control coordinates simultaneous transactions to avoid conflicts like lost updates and dirty reads.
