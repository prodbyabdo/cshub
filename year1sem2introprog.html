Lecture 1 · Intro to Computers & Hardware
Definition
A computer is a machine that performs a variety of tasks according to specific instructions. It's capable of performing computations and making logical decisions at high speeds.
6 Logical Hardware Units
  • Input Unit: Allows the user to enter information (keyboard, mouse).
  • Output Unit: Makes processed info available (screens, printers).
  • Memory Unit: Stores info. RAM (volatile) and ROM (non-volatile).
  • ALU: Arithmetic and Logic Unit performs calculations.
  • CPU: Central Processing Unit supervises overall operations.
  • Secondary Storage: Permanent storage (magnetic disks, CD-ROMs).
Lecture 2 · Computer Software
Programming Languages & Software Types

Low-Level Languages

Machine Language: 1s and 0s. Lowest level.
Assembly Language: Uses symbolic names for opcodes and operands. Needs translation.

High-Level Languages

Easier to understand. Uses compilers (translates complete unit) or interpreters (translates statement by statement). Ex: C++, Java.

Software Types

Application: Written to perform particular user tasks.
System: Operates the computer (e.g. Operating Systems like Windows/UNIX).

Lecture 3 · Program Development Life Cycle
The 4 Steps of PDLC
  1. Problem Definition: Specification of inputs/outputs and constraints.
  2. Problem Analysis: Breaking the problem into smaller, simpler sub-problems.
  3. Algorithm design: Clear specification of steps (Pseudocode/Flowchart).
  4. Coding & Debugging: Writing the source code and fixing errors (bugs).
Debugging & Variables
  • Compile-time error: Syntax error. Code won't compile into an executable.
  • Runtime error: Logic error (like an infinite loop). Program crashes while running.

Variables vs Constants: A constant never changes its value, while a variable's value changes during program execution. Names must start with a letter, have no spaces, and no hyphens.

  Statement

Lectures 7 & 8 · Repetition, Accumulator & Counter
Loop Constructs
  • Pre-Test Loop (While): Tests condition before execution. May execute 0 times.
  • Post-Test Loop (Do-While): Tests condition after execution. Always runs at least once.
  • Counted Loop (For): Executes a specified number of times using a control variable.
Counters & Accumulators
  • Counting: Used to keep track of iterations. Incrementing or decrementing by a set value (e.g. count = count + 1).
  • Accumulating: Summarizes numbers by adding a variable value to a running total on each pass (e.g. total = total + score).
