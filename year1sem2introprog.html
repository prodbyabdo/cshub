§1 · Java Platform — JDK, JRE, JVM & Compilation

What is Java?

Java is a high-level, object-oriented language released by Sun Microsystems in 1995, now owned by Oracle. It runs on Windows, macOS, Linux, and many other platforms. Over 3 billion devices run Java today.

The Three Layers

  • JDK: Java Development Kit. Everything a developer needs: compiler (javac), debugger, libraries, and the JRE. You install this to write and compile Java programs.
  • JRE: Java Runtime Environment. The part needed to run already-compiled Java programs. Includes the JVM + standard libraries. End-users install this; developers install the JDK (which contains it).
  • JVM: Java Virtual Machine. An abstract machine that reads bytecode and executes it on your actual hardware. This is why Java is platform-independent — bytecode is the same everywhere; the JVM adapts it per OS.

The Compilation Pipeline

Unlike C++ (which compiles directly to machine code), Java compiles to an intermediate format called bytecode, then the JVM interprets/JIT-compiles bytecode at runtime.

Source Code (Main.java) → Compiler (javac) → Bytecode (Main.class) → JVM (java Main) → Output (on any OS)

C++ vs Java compilation
In C++: g++ main.cpp -o main → produces a binary for that specific OS/CPU.
In Java: javac Main.java → produces Main.class (bytecode). Run anywhere with java Main.

Why Java? (from the lecture)

  • Platform-free: Write once, run anywhere — bytecode runs on any JVM
  • Popular: One of the most in-demand languages globally
  • OOP: Object-oriented design gives clear program structure and reuse
  • Open-source: Free to download and use
  • Secure & fast: Built-in security model; JIT compilation keeps it fast
  • Huge ecosystem: Tens of millions of developers; massive library support
  • Close to C++/C#: Syntax is familiar — easy to switch between the three

Java Editions

  • Java SE: Standard Edition — desktop & server apps. What this course uses.
  • Java EE: Enterprise Edition — web applications at scale
  • Java ME: Micro Edition — mobile/embedded devices
  • JavaFX: Rich internet application UIs

Key difference to memorise: Java uses String (capital S) as an object type, not a primitive. Always use .equals() to compare strings — never ==. The == operator on String objects checks if they are the same object in memory, not whether they have the same content.

§3 · Hello World & Program Structure

Every Java program lives inside a class. The class name must match the filename exactly. Execution begins inside main().

Lecture Example · Hello World

public class Main {
    public static void main(String[] args) {
        System.out.println("Hello World");
    }
}

Output: Hello World

  • public class Main: Declares a class named Main. File must be saved as Main.java. Java is case-sensitive.
  • public static void main: The entry point. Every Java program requires this exact signature. static means it belongs to the class itself, not an instance.
  • String[] args: Command-line arguments passed in. Required syntax even if unused.
  • System.out.println(): System is a built-in class. out is the output stream. println prints a line and adds a newline character.
  • Semicolon ; Every statement ends with ; — same as C++.
  • Curly braces {} Mark the start and end of code blocks — same as C++.
§4 · Output — println, print, numbers

Multiple println calls

System.out.println("Hello World!");
System.out.println("I am learning Java");
System.out.println("It is awesome!");

Output:
Hello World!
I am learning Java
It is awesome!

print() vs println() — no newline

System.out.print("Hello World! ");
System.out.print("I will print on the same line.");

Output: Hello World! I will print on the same line.

Print numbers

System.out.println(3);
System.out.println(358);
System.out.println(50000);

Maths inside println

System.out.println(3 + 3);   // 6
System.out.println(2 * 5);   // 10
System.out.println(10 / 2); // 5

Double quotes rule: Text must always be wrapped in double quotes. System.out.println(Hello); causes a compile error — Java treats Hello as a variable name, not text. Always write "Hello".

§5 · Comments

Single-line comment

// This is a comment
System.out.println("Hello World"); // this prints Hello World

Multi-line comment

/* The code below will print the words Hello World
   to the screen, and it is amazing! */
System.out.println("Hello World");

Rule of thumb: Use // for short inline notes. Use /* */ for longer explanations or temporarily disabling a block of code.

§6 · Variables & Data Types

Variables are containers for storing data values. All variables in Java must be declared with a type before use. Syntax: type variableName = value;

Declaring variables — all types

int myNum = 5;
float myFloatNum = 5.99f;
char myLetter = 'D';
boolean myBool = true;
String myText = "Hello";

Declare now, assign later

int myNum;
myNum = 15;
System.out.println(myNum); // 15

Overwriting a variable

int myNum = 15;
myNum = 20;                 // overrides the previous value
System.out.println(myNum); // 20

Final (constant) variable

final int myNum = 15;
myNum = 20; // error: cannot assign a value to a final variable

Display variables with text

String name = "John";
System.out.println("Hello, my name is: " + name);

String firstName = "John ";
String lastName = "Doe";
System.out.println(firstName + lastName); // John Doe

int x = 5;
int y = 6;
System.out.println(x + y); // 11 (math, not concat)

Multiple variables

int x = 5, y = 6, z = 50;
System.out.println(x + y + z); // 61

int a, b, c;
a = b = c = 50;
System.out.println(a + b + c); // 150

§7 · Type Casting

Type casting converts a value from one type to another. Java has two kinds:

  • Widening: Smaller → larger type. Happens automatically. No data loss. byte → short → char → int → long → float → double
  • Narrowing: Larger → smaller type. Must be done manually. Risk of data loss.

Widening casting — automatic

int myInt = 9;
double myDouble = myInt; // automatic: int → double
System.out.println(myInt);    // 9
System.out.println(myDouble); // 9.0

Narrowing casting — manual

double myDouble = 9.78d;
int myInt = (int) myDouble; // manual: double → int (truncates)
System.out.println(myDouble); // 9.78
System.out.println(myInt);    // 9

Real-life — percentage score

int score = 90;
int maxScore = 100;
float percentage = (float) score / maxScore * 100;
System.out.println("Your score is: " + percentage + "%");
// Output: Your score is: 90.0%

§9 · Strings & String Methods

In Java, String is an object (not a primitive). It has built-in methods you can call on it using dot notation.

String length

String txt = "ABCDEFGHIJKLMNOPQRSTUVWXYZ";
System.out.println("The length of the txt string is: " + txt.length());
// Output: The length of the txt string is: 26

toUpperCase / toLowerCase

String txt = "Hello World";
System.out.println(txt.toUpperCase()); // HELLO WORLD
System.out.println(txt.toLowerCase()); // hello world

String concatenation

String firstName = "John";
String lastName = "Doe";
System.out.println(firstName + " " + lastName); // John Doe

// using concat()
String first = "John ";
String last = "Doe";
System.out.println(first.concat(last)); // John Doe

Common String methods summary:
s.length() — character count
s.toUpperCase() — all caps
s.toLowerCase() — all lower
s.charAt(i) — character at index i
s.equals("x") — compare content (not ==)
s.indexOf("x") — find position of substring

§10 · Conditions & Branching

Simple if

int x = 20;
if (x > 18) {
    System.out.println("x is greater than 18");
}

if-else

int time = 20;
if (time < 18) {
    System.out.println("Good day.");
} else {
    System.out.println("Good evening.");
}
// Output: Good evening.

else if

int time = 22;
if (time < 10) {
    System.out.println("Good morning.");
} else if (time < 18) {
    System.out.println("Good day.");
} else {
    System.out.println("Good evening.");
}
// Output: Good evening.

Real-life example — door code

int doorCode = 1337;
if (doorCode == 1337) {
    System.out.println("Correct code. The door is now open.");
} else {
    System.out.println("Wrong code. The door remains closed.");
}

Switch statement — weekday

int day = 4;
String dayName;
switch (day) {
    case 1:  dayName = "Monday";    break;
    case 2:  dayName = "Tuesday";   break;
    case 3:  dayName = "Wednesday"; break;
    case 4:  dayName = "Thursday";  break;
    case 5:  dayName = "Friday";    break;
    case 6:  dayName = "Saturday";  break;
    case 7:  dayName = "Sunday";    break;
    default: dayName = "Invalid";
}
System.out.println(dayName); // Thursday

Don't forget break! Without break, Java "falls through" into the next case and keeps executing. Always end each case with break; unless fall-through is intentional.

§11 · Loops

while loop

int i = 0;
while (i < 5) {
    System.out.println(i);
    i++;
}
// Output: 0 1 2 3 4 (each on new line)

do-while loop

int i = 0;
do {
    System.out.println(i);
    i++;
} while (i < 5);
// Runs at least once — even if i >= 5 from the start

for loop — 0 to 4

for (int i = 0; i < 5; i++) {
    System.out.println(i);
}
// Output: 0 1 2 3 4

for loop — even numbers 0–10

for (int i = 0; i <= 10; i += 2) {
    System.out.println(i);
}
// Output: 0 2 4 6 8 10

Nested loops

// Outer loop
for (int i = 1; i <= 2; i++) {
    System.out.println("Outer: " + i);
    // Inner loop
    for (int j = 1; j <= 3; j++) {
        System.out.println("  Inner: " + j);
    }
}

Output: Outer: 1   Inner: 1   Inner: 2   Inner: 3   Outer: 2   Inner: 1   Inner: 2   Inner: 3

break — exit loop early

for (int i = 0; i < 10; i++) {
    if (i == 4) {
        break; // stop the loop when i equals 4
    }
    System.out.println(i);
}
// Output: 0 1 2 3

continue — skip one iteration

for (int i = 0; i < 10; i++) {
    if (i == 4) {
        continue; // skip 4, keep going
    }
    System.out.println(i);
}
// Output: 0 1 2 3 5 6 7 8 9

Real-life countdown

int countdown = 5;
while (countdown > 0) {
    System.out.println(countdown);
    countdown--;
}
System.out.println("Go!");
// Output: 5 4 3 2 1 Go!
