## Algorithm

```
’üČ An Algorithm is a clear and unambiguous specification of the steps needed to
solve a problem. It may be expressed in either Human language (English,
Tagalog), through a graphical representation like a flowchart or through a pseudo
code, which is a cross between human language and a programming language.
```
```
’üČ There are two commonly used tools to help to document program logic (the
algorithm).
’āś Flowcharts
’āś Pseudo code
```
```
’üČ Generally, flowcharts work well for small problems but Pseudo code is used for
larger problems.
```
## Pseudo Code

```
’üČ Pseudo code is a nonstandard English-like programming language used by
programmers to model logic and instructions.
```
```
’üČ Pseudo-Code is simply a numbered list of instructions to perform some task.
```
```
’üČ Pseudo code is called a programming language even through it cannot be
excluded by a computer.
```
```
’üČ It is designed for planning purposes only. By nonstandard, if you open five books
on programming, all would have pseudo code examples but all of those examples
would look somewhat different.
```
```
’üČ Most computer languages have required standards that must be followed when
creating programs.
```
```
’üČ Pseudo code provides us with a way to concentrate on just logic structures
without be constrained by the syntax rules of a particular programming language.
```
